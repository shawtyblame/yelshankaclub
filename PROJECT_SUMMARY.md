# Проект: Интернет-магазин автосервиса

## 1. Назначение

Веб-приложение для автосервиса / магазина автозапчастей. Позволяет:
- Просматривать каталог услуг и запчастей
- Оформлять заказы на запчасти с доставкой или самовывозом
- Записываться на установку / сервисное обслуживание
- Управлять заказами через админ-панель
- Регистрироваться и отслеживать историю заказов

## 3. Архитектура

### SPA (Single Page Application)
Фронтенд и бэкенд работают на одном домене:
- **В разработке**: Vite Dev Server (порт 5173) проксирует `/api` на Express (порт 3001)
- **В продакшене**: Express сам раздаёт статику (`dist/`) и обрабатывает API

```
Браузер → Express (порт 3001)
            ├── Статика: / → dist/index.html
            ├── API: /api/* → маршруты сервера
            └── Изображения: /uploads/*
```

### Маршруты React Router:
- `/` — Главная (Hero, Каталог, Услуги, Галерея)
- `/catalog` — Каталог запчастей
- `/services` — Услуги
- `/product/:id` — Детальная страница товара
- `/service/:id` — Детальная страница услуги
- `/cart` — Корзина (оформление заказа)
- `/auth` — Вход / Регистрация
- `/profile` — Личный кабинет (история заказов, смена пароля)
- `/admin` — Админ-панель
- `/feedback` — Форма обратной связи
- `/consumer-info` — Информация для потребителей (ПД, оплата, возврат)
- `/gallery` — Галерея работ
- `/gallery/:id` — Детальная страница работы

## 4. База данных (PostgreSQL)

### Таблицы:

- **users** — пользователи (id, email, password, name, avatar, role)
- **services** — услуги автосервиса (id, name_ru, name_en, description, category, price, image)
- **parts** — запчасти (id, name_ru, name_en, description, category, price, image, stock)
- **bookings** — заказы и записи (id, user_id, name, phone, email, car, service, date, time, message, status)
- **gallery** — галерея работ (id, title_ru, title_en, description, image)
- **gallery_comments** — комментарии к изображениям в галерее
- **contacts** — контактные данные (тип, значение, метка)

## 5. API endpoints

### Публичные:
- `GET /api/services` — список услуг
- `GET /api/services/:id` — детально об услуге
- `GET /api/parts` — список запчастей
- `GET /api/parts/:id` — детально о запчасти
- `GET /api/gallery` — галерея
- `GET /api/gallery/:id` — элемент галереи + комментарии
- `GET /api/contacts` — контакты
- `POST /api/bookings` — создание заказа (без авторизации)
- `POST /api/feedback` — отправка сообщения (через email)
- `POST /api/auth/login` — вход
- `POST /api/auth/register` — регистрация

### Требуют авторизации:
- `GET/PUT /api/auth/me` — профиль
- `PUT /api/auth/password` — смена пароля
- `PUT /api/auth/profile` — обновление профиля
- `GET /api/bookings/my` — история заказов пользователя

### Админ-панель (требуют роль admin):
- CRUD `/api/services`, `/api/parts`, `/api/gallery`, `/api/contacts`
- `GET/PUT/DELETE /api/bookings` — управление заказами
- `PUT /api/parts/:id/stock` — управление остатками
- `POST /api/upload` — загрузка изображений

## 6. Особенности реализации

### Авторизация:
- JWT токен (7 дней), хранится в localStorage
- Админ создаётся автоматически при инициализации БД
- Роли: user / admin

### Мультиязычность:
- i18next + react-i18next
- Два языка: русский, английский
- Переключение через Header

### Корзина:
- Контекст (CartContext) — глобальное состояние
- Добавление/удаление/изменение количества
- Автоматический пересчёт итога

### Оформление заказа:
- Выбор способа получения: самовывоз или доставка (с адресом)
- **Способ оплаты**: только "Наличные" активен, карта/перевод показаны как недоступные (скоро)
- **Согласие на обработку ПД** — обязательный чекбокс с ссылкой на политику
- Возможность записаться на установку вместе с заказом
- Выбор даты/времени для сервиса

### Форма обратной связи:
- Отправляет email через Nodemailer
- Чекбокс согласия на обработку ПД (обязательный)

### Регистрация:
- Стандартная форма с email/паролем/именем
- Чекбокс согласия на обработку ПД (обязательный)

### Страница "Информация для потребителей" (/consumer-info):
- 6 разделов: исполнитель, услуги, товары, оплата/доставка, ПД, возврат/претензии
- Ссылка в футере

## 7. Деплой

- **Хостинг**: Railway.app
- **Способ**: авто-деплой с GitHub (push → автоматический билд и деплой)
- **Конфигурация**: railway.json (Nixpacks builder)
- **Build**: npm install → npm run build
- **Start**: node server/index.cjs
- **БД**: Railway PostgreSQL, подключение через DATABASE_URL

### Переменные окружения:
- `DATABASE_URL` — строка подключения к PostgreSQL (внутренняя)
- `DATABASE_PUBLIC_URL` — публичная строка подключения (для локальных скриптов)
- `JWT_SECRET` — секрет для подписи токенов
- `SMTP_USER` / `SMTP_PASS` — почтовые учётные данные
- `PORT` — порт сервера
