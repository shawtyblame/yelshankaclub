# Техническое задание
## Yelshanka Club — Система управления автосервисом

---

## 1. Введение

### 1.1 Наименование системы
**Yelshanka Club** — веб-система для управления автосервисом премиум-класса.

### 1.2 Назначение системы
Автоматизация процессов записи клиентов на услуги тюнинга, управления каталогом услуг и обработки заявок через административную панель.

### 1.3 Планируемые сроки
Разработка: 4 недели

---

## 2. Функциональные требования

### 2.1 Публичная часть (Frontend)

#### 2.1.1 Главная страница (`/`)
| Элемент | Описание | Приоритет |
|---------|----------|-----------|
| Header | Логотип, навигация (Запись, Контакты), кнопка входа, переключатель языка | Высокий |
| Hero-секция | Полноэкранный градиентный баннер, заголовок с градиентным текстом, CTA-кнопка "Записаться на сервис" | Высокий |
| Форма записи | Поля: имя, телефон, email, марка авто, тип услуги (dropdown), дата, сообщение. Валидация в реальном времени | Высокий |
| Блок контактов | Телефон, email, адрес, ссылки на соцсети | Средний |
| Footer | Копирайт, контакты, навигация | Низкий |

#### 2.1.2 Страница аутентификации (`/auth`)
- Форма входа (email, пароль)
- Форма регистрации (имя, email, пароль, подтверждение пароля)
- Переключение между входом/регистрацией
- Валидация полей
- Отображение ошибок авторизации

#### 2.1.3 Административная панель (`/admin`)
Только для пользователей с ролью `admin`.

| Раздел | Функции |
|--------|---------|
| Управление услугами | Создание, редактирование, удаление услуг |
| Управление заявками | Просмотр списка, изменение статуса (новая → подтверждена → завершена) |
| Навигация | Кнопка возврата на главную |

### 2.2 Программный интерфейс (API)

#### 2.2.1 Аутентификация
| Метод | Endpoint | Описание |
|-------|----------|----------|
| POST | `/api/auth/register` | Регистрация нового пользователя |
| POST | `/api/auth/login` | Авторизация, возврат JWT |
| GET | `/api/auth/me` | Получение данных текущего пользователя |

#### 2.2.2 Услуги
| Метод | Endpoint | Описание | Доступ |
|-------|----------|----------|--------|
| GET | `/api/services` | Список всех услуг | Публичный |
| GET | `/api/services/:id` | Одна услуга по ID | Публичный |
| POST | `/api/services` | Создание услуги | Admin |
| PUT | `/api/services/:id` | Обновление услуги | Admin |
| DELETE | `/api/services/:id` | Удаление услуги | Admin |

#### 2.2.3 Заявки (Bookings)
| Метод | Endpoint | Описание | Доступ |
|-------|----------|----------|--------|
| POST | `/api/bookings` | Создание заявки | Публичный |
| GET | `/api/bookings` | Все заявки | Admin |
| GET | `/api/bookings/my` | Мои заявки | Авторизованный |
| PUT | `/api/bookings/:id/status` | Изменение статуса | Admin |

### 2.3 Структура данных

#### Пользователь (users)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT |
| name | TEXT | NOT NULL |
| email | TEXT | UNIQUE, NOT NULL |
| password | TEXT | NOT NULL (bcrypt hash) |
| role | TEXT | DEFAULT 'user' |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP |

#### Услуга (services)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT |
| name_ru | TEXT | NOT NULL |
| name_en | TEXT | NOT NULL |
| description_ru | TEXT | |
| description_en | TEXT | |
| category | TEXT | мощность, выхлоп, подвеска, экстерьер, интерьер, другое |
| price | REAL | NOT NULL |
| image_url | TEXT | |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP |

#### Заявка (bookings)
| Поле | Тип | Ограничения |
|------|-----|-------------|
| id | INTEGER | PRIMARY KEY, AUTOINCREMENT |
| name | TEXT | NOT NULL |
| phone | TEXT | NOT NULL |
| email | TEXT | NOT NULL |
| car_brand | TEXT | NOT NULL |
| service_id | INTEGER | FOREIGN KEY → services |
| booking_date | TEXT | NOT NULL |
| message | TEXT | |
| status | TEXT | DEFAULT 'new' (new/confirmed/completed) |
| user_id | INTEGER | FOREIGN KEY → users (nullable) |
| created_at | TEXT | DEFAULT CURRENT_TIMESTAMP |

## 3. Нефункциональные требования

### 3.1 Производительность
- Время отклика страницы < 2 секунд
- Время ответа API < 500 мс
- Одновременных пользователей: до 100

### 3.2 Безопасность
- Пароли хешируются через bcrypt (cost factor 10)
- JWT-токены сроком действия 24 часа
- Валидация всех входных данных на сервере
- Защита от SQL-инъекций (параметризованные запросы)
- CORS настроен только для localhost

### 3.3 Совместимость
- Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- Адаптивный дизайн: Desktop (1200px+), Tablet (768-1199px), Mobile (<768px)

### 3.4 Локализация
- Поддержка русского и английского языков
- Переключение без перезагрузки страницы
- Сохранение выбора в localStorage

## 7. Релиз и деплой

### 7.1 Локальная разработка
```bash
npm run dev        # Frontend (5173)
npm run dev:server # Backend (3001)
npm run dev:all    # Оба
```

### 7.2 Сборка для продакшена
```bash
npm run build      # SPA в dist/
node server/index.cjs  # Запуск сервера
```

### 7.3 Переменные окружения
| Переменная | Описание | Значение по умолчанию |
|------------|----------|----------------------|
| PORT | Порт сервера | 3001 |
| JWT_SECRET | Секрет для JWT | yelshanka-secret-key |
| DB_PATH | Путь к SQLite | ./database.sqlite |
